  • Vancomer

    This is a full stack website which intend to help people that is new to Vancouver. Our team came up with this idea because we all faced different difficulties as an international student. There are 2 types of users in this website: the client role and the job seeker role. Clients, who are new to Vancouver, can post requests on our platform and get help with differnt issues like housing, transportation, education, etc. Clients specify the time for this job/request. These jobs/requests are store in our database. A job seeker can signup and specify the predefined areas that they are able to provide help. When they login, their dashboard page will fetch jobs/requests from our database, according to the areas that the job seekers declared when they signup. They can apply for the job/request and provide the amount of money needed for this job. The client can see all the applicants on the job they posted and choose their provider.

    There are a few issues in this project right now. First, the database will sleep while not using. It also takes sometime to wake up. This is because we have not paid for the database to be up 24/7. Second, we planned to update the website with signin with social media features in the future. Lastly, we will also look into the payment system if we decided to actually run this in the real world. Since maintance have a cost, we may collect 50 cents, a dollar, or a small percentage of the job charged by the job seeker on each transaction.

    Additionally, if we are actually launching this website, we will also add other security related features like 2 step authentication and more secure data encryption.

  • EasyGrammar

    This is a front-end website that uses the FreeDictionary API and OpenAI API to assist our user to improve their English. User starts to use the website by entering a word. The website will use the user's input and fetch data from FreeDictionary API. If the fetch is successful, it will display the word's definition and an pronounciation button which plays an audio of how the word is pronounced. Otherwise, it will display the word the user entered is not found. After the definition is displayed, it will also fetch an image generated by the OpenAI API that is related to the searched word. The user then need to describe the image as detail as possible. The website will send the user's description with a evaluation process to OpenAI API for an grammar analysis and a grade for the user. In this way, the user can practice English writing and learn new vocabularies to improve their English.
